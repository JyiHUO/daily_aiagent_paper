早上好，今天是 2026 年 04 月 07 日。这期 AI Agent 论文播报，主要覆盖 cs.AI、cs.CL、cs.LG、cs.MA、cs.RO、cs.SE、cs.HC 这些分类里最值得看的工作。

今天初筛后最集中的主题是 general_agent、agent_safety、embodied_agent，说明研究关注点继续从单轮回答能力转向更完整的执行链。

高优先级论文里，GeoBrowse: A Geolocation Benchmark for Agentic Tool Use with Expert-Annotated Reasoning Traces 等工作都在强调 Agent 的系统边界，而不是只卷更大的底座模型。

从创新性和研究开拓性看，GeoBrowse: A Geolocation Benchmark for Agentic Tool Use with Expert-Annotated Reasoning Traces、Full-Duplex-Bench-v3: Benchmarking Tool Use for Full-Duplex Voice Agents Under Real-World Disfluency 代表了今天最值得后续继续追踪的切口。

先补一个今天特别值得懂的基础概念，叫做Prompt Injection 是什么。Prompt Injection 本质上是把原本应该当作数据读取的内容，偷偷变成了会影响模型行为的指令。它的危险不只在用户直接输入恶意话术，更在网页、邮件、文档、工具返回结果等外部内容里夹带隐藏命令。对 Agent 来说，一旦系统会跨多步读取环境并调工具，注入就会从'输出脏话'升级成'做错动作'，所以它是执行链安全里的基础概念。 今天的重点论文里，反复出现了评测、执行链安全、工具与环境交互这些问题，所以补这个概念最划算。

第 1 篇论文是《Your Agent is More Brittle Than You Think: Uncovering Indirect Injection Vulnerabilities in Agentic LLMs》。它的主题是 agent_safety。为什么入选？这篇论文直接研究 Agentic LLM 在真实多步工具调用里的间接提示注入问题，不是只看单轮问答里的'越狱'。它的价值在于：一方面系统比较了4类攻击、6类防御、9个模型骨干，证明很多常见防御在动态环境里几乎挡不住；另一方面提出把检测点前移到模型的潜在表示层，在工具输入阶段就做'断路器'，对 Agent 产品落地很有现实指导意义。。这篇论文的背景是：很多 Agent 安全评测还停留在单轮、静态文本场景，但真实 Agent 会读外部内容、调工具、跨多步执行，攻击面大很多。论文关注的是间接提示注入：恶意指令藏在网页、交易记录、邮件等第三方内容里，Agent 在正常流程中把它当作上下文读进去，随后自己执行危险动作。作者发现，传统的表层防御如提示警告、改写、关键词过滤、LLM 审查，在这种动态流程里普遍不稳，甚至可能带来副作用，因此现在非常值得看。。如果只记一个关键信号，我会记它的创新性分数是 46，研究开拓性分数是 52，说明它不仅有眼前价值，也可能影响后续 Agent 研究路线。

第 2 篇论文是《GUIDE: Interpretable GUI Agent Evaluation via Hierarchical Diagnosis》。它的主题是 agent_eval。为什么入选？这篇论文很值得放进 Agent 日报，因为它不是再做一个 GUI Agent，而是解决 GUI Agent '怎么评、怎么定位错因' 这个更接近产品落地的问题。论文把原来一次性给整条轨迹下结论的黑盒评测，改成先切分子任务、再逐段诊断、最后汇总，既提高了长轨迹评测准确率，又能直接产出可操作的错误报告。对做 computer use、网页代理、手机代理的团队来说，这类可解释评测比单一成功率更能指导迭代。。这篇论文的背景是：GUI Agent 的任务往往步骤长、依赖视觉状态，而且同一个目标可能有多条可行路径，所以评测很难既准确又能解释原因。现有很多方法用一次 LLM 调用对整条轨迹做整体判断，轨迹一长就容易被上下文拖垮，而且最后只给出成功或失败，几乎不能回答'错在哪一步、为什么错、该怎么改'。这篇论文值得关注，是因为它把分层分解思想从 Agent 执行侧搬到了评测侧，在不做任务规则工程、也不额外训练模型的前提下，显著提升了长轨迹 GUI Agent 的评测鲁棒性和可诊断性。。如果只记一个关键信号，我会记它的创新性分数是 50，研究开拓性分数是 45，说明它不仅有眼前价值，也可能影响后续 Agent 研究路线。

第 3 篇论文是《The Tool Illusion: Rethinking Tool Use in Web Agents》。它的主题是 web_agent。为什么入选？这篇论文直接挑战了一个在 Web Agent 里很流行的默认前提：'上工具一定更强'。作者不是只看单一框架或单一模型，而是把人写工具、模型合成工具、不同 backbone、不同 benchmark 放到同一控制条件下重测，结论对 Agent 设计和评测都很有现实价值。。这篇论文的背景是：Web Agent 过去常靠点击、输入这类原子浏览器动作完成任务，但这种方式容易脆、步骤长、每一步都要重新推理。于是很多工作开始引入更高层的工具，比如 API 或把一串网页操作打包成函数，希望一键替代多步操作。论文指出，已有结论往往来自小规模、不可直接比较的实验，因此'工具是否真的普遍有效、什么样的工具才好、工具会带来什么副作用'其实并不清楚。这篇工作值得看，因为它把这个问题系统地重新做了一遍，而且不少结果会直接影响 Web Agent 的产品架构和评测口径。。如果只记一个关键信号，我会记它的创新性分数是 50，研究开拓性分数是 63，说明它不仅有眼前价值，也可能影响后续 Agent 研究路线。

如果把今天的论文连起来看，一个明显变化是：大家越来越少把 Agent 当成单一模型能力问题，而是把它当成执行链、工具层、评测层和安全层共同构成的系统问题。

这意味着后续真正有开拓性的研究，往往不是再加一点 prompt 技巧，而是重新定义 Agent 应该如何被评测、如何被约束，以及如何在真实环境里更稳地工作。
